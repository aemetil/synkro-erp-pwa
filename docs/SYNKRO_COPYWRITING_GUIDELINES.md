# Synkro Copywriting Guidelines v2

> Direction : **clair, calme, professionnel, humain**  
> Objectif : définir le langage produit de Synkro v2 pour renforcer la confiance, réduire la confusion et accompagner les utilisateurs non techniques.

---

## 1. Pourquoi ce document existe

Synkro n’est pas seulement une interface.

Synkro parle à l’utilisateur à travers :

- les titres ;
- les boutons ;
- les messages d’erreur ;
- les confirmations ;
- les empty states ;
- les formulaires ;
- les reçus ;
- les rapports ;
- les notifications ;
- les messages WhatsApp.

Le langage doit donc être cohérent, simple et rassurant.

Un bon wording peut éviter beaucoup de confusion.

---

## 2. Principe central

> Synkro doit parler comme un assistant calme, pas comme un système technique.

Le produit doit toujours aider l’utilisateur à comprendre :

```txt
Où suis-je ?
Qu’est-ce que je peux faire ?
Qu’est-ce qui vient de se passer ?
Qu’est-ce que je dois faire maintenant ?
```

---

## 3. Ton de voix

Le ton Synkro doit être :

- clair ;
- sobre ;
- humain ;
- professionnel ;
- direct ;
- rassurant ;
- non agressif ;
- non infantilisant.

---

## 4. Ce qu’il faut éviter

Éviter :

```txt
Oops!
Boostez votre business !
Une erreur inconnue est survenue.
Server action failed.
CredentialsSignin.
Invalid input.
Transaction mutated successfully.
Êtes-vous sûr ?
Aucune donnée.
```

Ces messages sont soit trop techniques, soit trop génériques, soit trop marketing.

---

## 5. Ce qu’il faut préférer

Préférer :

```txt
Impossible d’enregistrer pour le moment.
Vérifiez votre connexion puis réessayez.

Email ou mot de passe incorrect.

Vente enregistrée.

Paiement ajouté.

Aucune vente pour le moment.
Ajoutez votre première vente pour commencer à suivre votre activité.
```

---

## 6. Règles générales d’écriture

### 6.1 Être concret

Mauvais :

```txt
Action réussie.
```

Bon :

```txt
Vente enregistrée.
```

Mauvais :

```txt
Erreur.
```

Bon :

```txt
Impossible d’enregistrer la vente.
```

---

### 6.2 Être court

Les utilisateurs ne doivent pas lire un paragraphe pour comprendre.

Mauvais :

```txt
Nous avons rencontré un problème lors du traitement de votre demande. Veuillez vérifier votre connexion internet ou contacter le support si le problème persiste.
```

Bon :

```txt
Impossible d’enregistrer.
Vérifiez votre connexion puis réessayez.
```

---

### 6.3 Dire quoi faire ensuite

Mauvais :

```txt
Échec.
```

Bon :

```txt
Impossible de charger cette page.
Vous pouvez réessayer.
```

---

### 6.4 Ne pas accuser l’utilisateur

Mauvais :

```txt
Vous avez entré un montant invalide.
```

Bon :

```txt
Le montant doit être supérieur à 0.
```

Mauvais :

```txt
Vous n’avez pas rempli ce champ.
```

Bon :

```txt
Ce champ est obligatoire.
```

---

### 6.5 Ne pas sur-vendre

Mauvais :

```txt
Révolutionnez votre business avec Synkro !
```

Bon :

```txt
Suivez vos ventes, dépenses et paiements plus simplement.
```

---

## 7. Langue principale

La langue principale actuelle recommandée est :

```txt
Français simple
```

À terme, prévoir :

```txt
Français
Créole haïtien
```

Le français doit rester accessible.

Éviter les termes trop comptables si un mot simple suffit.

## 9. Boutons

### 9.1 Règles

Les boutons doivent commencer par un verbe clair.

Préférer :

```txt
Nouvelle vente
Ajouter un paiement
Enregistrer
Créer le produit
Générer le reçu
Envoyer par WhatsApp
Inviter un employé
Réessayer
```

Éviter :

```txt
OK
Submit
Valider
Action
Continuer ici
Gestion
```

---

### 9.2 Boutons principaux

| Contexte               | Label recommandé              |
| ---------------------- | ----------------------------- |
| Créer vente            | Nouvelle vente                |
| Soumettre vente        | Enregistrer la vente          |
| Créer dépense          | Nouvelle dépense              |
| Soumettre dépense      | Enregistrer la dépense        |
| Créer produit          | Nouveau produit               |
| Soumettre produit      | Enregistrer le produit        |
| Ajouter paiement       | Ajouter le paiement           |
| Générer reçu           | Générer le reçu               |
| Partager WhatsApp      | Envoyer par WhatsApp          |
| Inviter utilisateur    | Inviter un employé            |
| Sauvegarder paramètres | Enregistrer les modifications |

---

### 9.3 Boutons secondaires

```txt
Annuler
Retour
Voir les détails
Voir l’historique
Modifier
Filtrer
Exporter
Plus tard
Ignorer pour l’instant
```

---

### 9.4 Boutons destructifs

Ils doivent être très explicites.

```txt
Supprimer la vente
Supprimer le produit
Annuler le paiement
Retirer l’utilisateur
```

Éviter :

```txt
Supprimer
Confirmer
Oui
```

## 12. Messages de succès

### 12.1 Règle

Le message doit confirmer précisément ce qui vient de se passer.

### 12.2 Exemples

```txt
Vente enregistrée.
Dépense enregistrée.
Produit mis à jour.
Stock ajusté.
Paiement ajouté.
Client mis à jour.
Paramètres enregistrés.
Utilisateur invité.
Reçu généré.
Signalement envoyé.
```

### 12.3 Succès avec prochaine action

Après paiement :

```txt
Paiement ajouté.
Le solde du client a été mis à jour.
```

Après reçu :

```txt
Reçu généré.
Vous pouvez maintenant l’envoyer au client.
```

Après première vente :

```txt
Votre première vente est enregistrée.
Synkro commence maintenant à suivre votre activité.
```

## 13. Messages d’erreur

### 13.1 Règle

Une erreur doit dire :

```txt
ce qui n’a pas fonctionné
quoi faire maintenant
```

### 13.2 Erreurs générales

```txt
Impossible de charger cette page.
Vous pouvez réessayer dans quelques instants.
```

```txt
Impossible d’enregistrer pour le moment.
Vérifiez votre connexion puis réessayez.
```

```txt
Une erreur est survenue.
Aucune modification n’a été appliquée.
```

---

### 13.3 Erreurs login

Mauvais :

```txt
CredentialsSignin
Configuration
```

Bon :

```txt
Email ou mot de passe incorrect.
```

```txt
Impossible de se connecter pour le moment.
Réessayez dans quelques instants.
```

---

### 13.4 Erreurs formulaire

```txt
Ce champ est obligatoire.
Le montant doit être supérieur à 0.
Veuillez choisir une date.
Veuillez choisir un client.
Veuillez choisir un produit.
Le stock disponible est insuffisant.
Le montant reçu dépasse le solde restant.
```

---

### 13.5 Erreurs permission

```txt
Vous n’avez pas l’autorisation d’effectuer cette action.
```

```txt
Seul un administrateur peut modifier ces paramètres.
```

```txt
Demandez au propriétaire de l’entreprise si vous avez besoin d’accès.
```

---

### 13.6 Erreurs réseau

```txt
Connexion instable.
Vos données peuvent prendre plus de temps à charger.
```

```txt
Vous êtes hors ligne.
Certaines actions ne sont pas disponibles pour le moment.
```

```txt
Connexion retrouvée.
Synchronisation en cours…
```

## 14. Confirmations

### 14.1 Règle

Les confirmations doivent expliquer l’impact.

Mauvais :

```txt
Êtes-vous sûr ?
```

Bon :

```txt
Supprimer cette vente ?
Cette action ne peut pas être annulée. Les montants liés à cette vente seront retirés des rapports.
```

---

### 14.2 Suppression vente

```txt
Supprimer cette vente ?
Cette action ne peut pas être annulée. Les montants liés à cette vente seront retirés des rapports.
[ Supprimer la vente ] [ Annuler ]
```

---

### 14.3 Suppression produit

```txt
Supprimer ce produit ?
Il ne sera plus disponible dans votre catalogue. L’historique existant restera conservé.
[ Supprimer le produit ] [ Annuler ]
```

---

### 14.4 Annulation paiement

```txt
Annuler ce paiement ?
Le paiement restera visible dans l’historique comme annulé.
[ Annuler le paiement ] [ Retour ]
```

---

### 14.5 Changement de devise

```txt
Changer la devise principale ?
Cette devise sera utilisée pour les nouveaux affichages. Les montants existants ne seront pas automatiquement convertis.
[ Changer la devise ] [ Annuler ]
```


```txt

```

```txt
Il reste 3 200 G à encaisser.
```

```txt
Vos dépenses du mois sont de 4 500 G.
```

```txt

```

### 18.2 Éviter

```txt
Revenue
Net profit
Cashflow delta
```

Sauf si ces termes sont expliqués.

## 22. WhatsApp

### 22.1 Règles

Les messages WhatsApp doivent être :

- courts ;
- professionnels ;
- sans jargon ;
- sans données sensibles inutiles.

---

### 22.2 Reçu paiement complet

```txt
Bonjour [Client], voici votre reçu de paiement pour la vente [SaleNumber].

Montant payé : [Amount]
Statut : Payé

Merci.
```

---

### 22.3 Reçu paiement partiel

```txt
Bonjour [Client], voici votre reçu de paiement pour la vente [SaleNumber].

Montant payé : [PaidAmount]
Reste à payer : [RemainingAmount]

Merci.
```

---

### 22.4 Rappel paiement

```txt
Bonjour [Client], petit rappel pour votre paiement en attente.

Montant restant : [RemainingAmount]
Référence : [SaleNumber]

Merci.
```

---

### 22.5 Santé

Pour le module santé, éviter de mettre le diagnostic ou les informations médicales dans un message WhatsApp par défaut.

Préférer :

```txt
Bonjour [Client], votre document est prêt.
Vous pouvez le consulter via le lien envoyé par le cabinet.
```

## 28. Créole haïtien — préparation future

À terme, Synkro pourra proposer une version créole.

Principe :

- ne pas traduire mot à mot ;
- rester naturel ;
- tester avec utilisateurs locaux ;
- garder les mots business compréhensibles.

Exemple futur :

Français :

```txt
Vente enregistrée.
```

Créole possible :

```txt
Vant lan anrejistre.
```

Français :

```txt
Reste à payer
```

Créole possible :

```txt
Rès pou peye
```

À valider avec locuteurs haïtiens avant production.

## 29. Anti-patterns copywriting

Éviter :

- jargon technique ;
- anglais mélangé sans raison ;
- messages trop longs ;
- humour dans les moments sensibles ;
- ton agressif ;
- culpabilisation ;
- termes comptables non expliqués ;
- “Oops” ;
- “Erreur inconnue” ;
- “Contactez l’administrateur” sans contexte.

---

## 30. Checklist avant validation d’un texte

Avant de valider un texte dans Synkro, vérifier :

```txt
Est-ce clair ?
Est-ce court ?
Est-ce humain ?
Est-ce professionnel ?
Est-ce que l’utilisateur sait quoi faire ?
Est-ce que le texte évite le jargon ?
Est-ce adapté au mobile ?
Est-ce rassurant ?
```

---

## 31. Phrase directrice

> Synkro doit parler simplement, surtout quand l’action est importante.

Le langage doit réduire la peur, pas ajouter du bruit.
